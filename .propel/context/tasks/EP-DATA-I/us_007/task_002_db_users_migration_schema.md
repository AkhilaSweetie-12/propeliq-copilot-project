## Task Overview
Generate the EF Core migration for the `users` table using `dotnet ef migrations add UsersEntity`. Verify and harden the generated migration `Up()` to add: `CREATE EXTENSION IF NOT EXISTS pgcrypto` (for `gen_random_uuid()` compatibility on PostgreSQL 16), the `users` table DDL with all required columns and types, a `CHECK` constraint on `role` (`IN ('Patient','Staff','Admin')`), a `CHECK` constraint on `status` (`IN ('Active','Inactive')`), and a `UNIQUE` index named `users_email_key` on the `email` column. Write the corresponding `Down()` rollback. Document the `PHI_ENCRYPTION_KEY` key rotation procedure for EC-2.

## Implementation Plan

1. **Enable `pgcrypto` in the migration `Up()`** — required for `gen_random_uuid()` on PostgreSQL 16:
   After generating the migration with `dotnet ef migrations add UsersEntity`, open the generated file and manually add at the top of `Up()`:
   ```csharp
   migrationBuilder.Sql("CREATE EXTENSION IF NOT EXISTS pgcrypto;");
   ```
   This is idempotent; safe to apply multiple times. Note: `gen_random_uuid()` is available natively in PostgreSQL 13+ without pgcrypto, but enabling pgcrypto is required for future `pgp_sym_encrypt` usage (AC-2 references it as an option) and is already present from us_003/us_004.

2. **Verify generated migration DDL** — the `Up()` must produce:
   ```sql
   CREATE TABLE users (
       user_id      UUID         NOT NULL DEFAULT gen_random_uuid(),
       email        TEXT         NOT NULL,
       password_hash TEXT        NOT NULL,
       role         TEXT         NOT NULL,
       status       TEXT         NOT NULL DEFAULT 'Active',
       created_at   TIMESTAMPTZ  NOT NULL,
       updated_at   TIMESTAMPTZ  NOT NULL,
       CONSTRAINT pk_users PRIMARY KEY (user_id)
   );
   ```
   EF Core generates this from the `OnModelCreating` configuration in task_001. Review the generated migration and confirm these columns match.

3. **Add CHECK constraints on `role` and `status`** (AC-1, AC-4) — EF Core does NOT generate CHECK constraints from enum converters automatically; add them manually after the `CreateTable` call in `Up()`:
   ```csharp
   migrationBuilder.Sql(
       "ALTER TABLE users ADD CONSTRAINT chk_users_role " +
       "CHECK (role IN ('Patient', 'Staff', 'Admin'));");

   migrationBuilder.Sql(
       "ALTER TABLE users ADD CONSTRAINT chk_users_status " +
       "CHECK (status IN ('Active', 'Inactive'));");
   ```
   And corresponding `Down()` drops:
   ```csharp
   migrationBuilder.Sql("ALTER TABLE users DROP CONSTRAINT IF EXISTS chk_users_role;");
   migrationBuilder.Sql("ALTER TABLE users DROP CONSTRAINT IF EXISTS chk_users_status;");
   ```

4. **Add UNIQUE index named `users_email_key`** (AC-1, EC-3) — EF Core generates this from `HasIndex(u => u.Email).IsUnique().HasDatabaseName("users_email_key")` configured in task_001. Verify the generated migration contains:
   ```sql
   CREATE UNIQUE INDEX users_email_key ON users (email);
   ```
   If EF Core generated a different name, update either the `HasDatabaseName` call in task_001 or rename the index in the migration file. The constraint name `users_email_key` must match what `UserRepository.cs` checks for in the `DbUpdateException` catch filter (task_001 step 5).

5. **Write the complete `Down()` rollback**:
   ```csharp
   protected override void Down(MigrationBuilder migrationBuilder)
   {
       migrationBuilder.Sql("ALTER TABLE users DROP CONSTRAINT IF EXISTS chk_users_role;");
       migrationBuilder.Sql("ALTER TABLE users DROP CONSTRAINT IF EXISTS chk_users_status;");
       migrationBuilder.DropTable(name: "users");
   }
   ```
   The UNIQUE index and PRIMARY KEY are dropped automatically by `DropTable`.

6. **Apply the migration and verify**:
   ```bash
   DATABASE_URL="Host=localhost;Port=5433;Database=patientaccess;Username=devuser;Password=devpassword" \
   PHI_ENCRYPTION_KEY="dev-only-phi-key-change-in-production-32chars" \
   JWT_SIGNING_KEY="dev-only-signing-key-change-in-production-min-32chars" \
   dotnet ef database update --project api/Api.csproj
   ```
   Then verify via `psql`:
   ```sql
   \d users
   SELECT constraint_name, constraint_type FROM information_schema.table_constraints
   WHERE table_name = 'users';
   ```
   Expected output: 1 PRIMARY KEY, 1 UNIQUE, 2 CHECK constraints, 7 columns.

7. **Create `key-rotation-runbook.md`** for EC-2 — `PHI_ENCRYPTION_KEY` rotation procedure:
   ```markdown
   # PHI_ENCRYPTION_KEY Rotation Runbook

   ## When to rotate
   - Scheduled: every 90 days per TR-006
   - Emergency: if key compromise is suspected

   ## Steps
   1. Generate a new 256-bit key: `openssl rand -base64 32`
   2. Store the new key securely (vault, GitHub Secret, etc.)
   3. Create a new EF Core migration: `dotnet ef migrations add RotatePhiEmailEncryptionKey`
   4. In the migration `Up()`, execute a raw SQL statement that reads each `email`,
      decrypts with the OLD key (passed as a migration parameter or env var `PHI_ENCRYPTION_KEY_OLD`),
      re-encrypts with the NEW key (`PHI_ENCRYPTION_KEY`), and updates the row.
      Process in batches of 500 rows to avoid long-running transactions.
   5. Apply the migration in a maintenance window: `dotnet ef database update`
   6. Remove `PHI_ENCRYPTION_KEY_OLD` from secrets
   7. Record the rotation date and migration name in the audit log

   ## Rollback
   - Apply the `Down()` migration (reverses the re-encryption using `PHI_ENCRYPTION_KEY_OLD`)
   - Restore the old key as `PHI_ENCRYPTION_KEY`

   ## Notes
   - Never store the old key alongside the new key in the same secret store entry
   - The `email` UNIQUE index remains valid across rotation because the new ciphertext for
     each email differs, but uniqueness is preserved (one email = one unique ciphertext under the new key)
   ```

8. **Validate constraint enforcement** — confirm the CHECK constraint blocks invalid role values:
   ```sql
   -- This should fail with: ERROR: new row for relation "users" violates check constraint "chk_users_role"
   INSERT INTO users (user_id, email, password_hash, role, status, created_at, updated_at)
   VALUES (gen_random_uuid(), 'test@test.com', 'hash', 'SuperAdmin', 'Active', NOW(), NOW());
   ```
