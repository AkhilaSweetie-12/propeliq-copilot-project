# User Story - US_019

## Story ID

- ID Format: US_019

## Story Title

- Session Inactivity Timeout with Expiry Warning Modal

## Description

- As an authenticated user, I want to receive a warning modal two minutes before my session expires due to inactivity so that I can extend my session and avoid losing unsaved work, while the system automatically logs me out if I take no action.

## Acceptance Criteria

1. **Given** an authenticated user is active on any authenticated screen (SCR-002 through SCR-021), **When** 13 minutes of inactivity elapse without any user interaction (mouse movement, keypress, or API call), **Then** the React SPA displays the session-expiry warning modal (UXR-503) with: (a) the message "Your session will expire in 2 minutes due to inactivity", (b) a visible 2-minute countdown timer decrementing in real time, (c) an "Extend Session" button (primary), and (d) a "Sign out" button (ghost); the modal is rendered as a full-viewport overlay blocking interaction with the underlying page.

2. **Given** the session-expiry warning modal is displayed, **When** the user clicks "Extend Session", **Then** the client calls `POST /api/auth/refresh` with the current refresh token to obtain a new 15-minute JWT access token; the modal is dismissed; the inactivity timer resets to 13 minutes from that moment; the user remains on the current screen without interruption.

3. **Given** the session-expiry warning modal is displayed, **When** the user clicks "Sign out" in the modal, **Then** the client calls `POST /api/auth/logout` to blocklist the JWT and delete the refresh token from Redis; the SPA navigates to the Login screen (SCR-001); the inactivity timer is cleared.

4. **Given** the session-expiry warning modal is displayed and the 2-minute countdown reaches zero (15 total minutes of inactivity), **When** no user action was taken, **Then** the client calls `POST /api/auth/logout` automatically (server-side the JWT TTL has also expired); the SPA navigates to the Login screen (SCR-001) and displays a toast notification "You were signed out due to inactivity"; any unsaved form state is cleared from memory.

5. **Given** any user interaction (mouse movement, click, keypress, or scroll) occurs on an authenticated screen, **When** the inactivity timer is running (< 13 minutes elapsed), **Then** the timer resets to 0; the warning modal does NOT appear; no API call is made as a result of the reset.

6. **Given** the JWT access token has a 15-minute TTL set at the server level (via `TR-005`), **When** the token expires and the client attempts to call any protected API endpoint without a valid token, **Then** the API returns HTTP 401 regardless of client-side timer state — confirming that the server-side JWT expiry is the authoritative enforcement mechanism; the client-side timer provides UX feedback only and cannot extend or bypass server-side expiry.

## Edge Cases

- What happens when the user opens multiple browser tabs and is active in one tab but idle in another? → The inactivity timer runs independently per tab; a tab that reaches 13 minutes of inactivity shows the modal in that tab; extending the session in one tab calls `POST /api/auth/refresh`, which issues a new JWT that is also used by other tabs on their next API call — session is preserved globally; the other tabs' timers are not automatically reset but will reset on next user interaction.
- What happens when the "Extend Session" `POST /api/auth/refresh` call fails (e.g., network error)? → The modal remains visible with an error message "Unable to extend session — check your connection"; the 2-minute countdown continues; if the countdown reaches zero, the client proceeds with the automatic logout flow; no silent security bypass is introduced.
- What happens if the user is mid-form entry (e.g., intake form) when the session expires with no extension? → The intake form auto-saves to the server every 60 seconds (UXR-004, handled in EP-002); on logout, a toast notifies "Unsaved changes may have been lost"; when the user re-authenticates, the last auto-saved draft is restored from the server.
