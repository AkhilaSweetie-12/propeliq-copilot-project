# User Story - US_031

## Story ID

- ID Format: US_031

## Story Title

- Staff Home Dashboard & Walk-In Entry Point

## Description

- As an authenticated Staff user, I want to see today's operational metrics and a queue preview the moment I log in, so that I can immediately assess the day's workload, identify high-risk appointments, and navigate to the walk-in booking or queue management flows in one click.

## Acceptance Criteria

1. **Given** a user with `role = Staff` authenticates and receives a valid JWT, **When** the SPA redirects post-login, **Then** it routes to `/staff` (SCR-011 Staff Home Dashboard); a user with `role = Patient` or `role = Admin` is NOT routed to `/staff` — the API returns HTTP 403 Forbidden for any `/api/queue/*` or `/api/appointments/walkin` request from non-Staff roles; the RBAC guard is enforced at API middleware level independently of SPA routing.

2. **Given** the Staff dashboard loads, **When** `GET /api/queue/today` returns data, **Then** four metric cards are rendered showing: (a) total appointments today, (b) patients arrived, (c) patients still scheduled, (d) count of High-risk appointments; all four values are sourced from the same `GET /api/queue/today` response payload to avoid multiple API calls; a loading skeleton is shown while the request is in-flight (UXR-501).

3. **Given** the Staff dashboard renders, **When** the page is fully loaded, **Then** a queue preview section displays the first 5 upcoming appointments for today ordered by `slot_time ASC`, showing: patient name (linked to SCR-016 patient 360° view), appointment time, provider name, current status badge, and a no-show risk badge (↓ Low / — Medium / ↑ High with text + colour + icon — never colour-alone); a "View full queue →" link navigates to SCR-014.

4. **Given** the Staff is on SCR-011, **When** the Staff clicks the "+ Add walk-in" primary button (`#btn-add-walkin`), **Then** the SPA navigates to the Patient Search screen (SCR-013) to begin the walk-in booking flow (UC-004 entry); the navigation does not require a page reload.

5. **Given** the Staff dashboard is rendered and the Staff clicks "↻ Refresh", **When** the refresh action fires, **Then** `GET /api/queue/today` is re-called and all four metric cards and the queue preview are updated in place with no full-page reload; a visual refresh indicator is shown during the in-flight request; `aria-live="polite"` is set on the queue preview container so screen-reader users are notified of updated content.

## Edge Cases

- What happens when a Staff user's JWT expires while they are on the dashboard (15-minute inactivity timeout)? → The SPA detects the 401 Unauthorized response from `GET /api/queue/today` on the next refresh cycle; the session expiry modal (from EP-001, us_019) is displayed prompting re-authentication; no dashboard data is shown in a stale state after expiry.
- What happens if `GET /api/queue/today` returns an empty list (no appointments scheduled today)? → All four metric cards show "0"; the queue preview section renders an empty-state message "No appointments scheduled today — add a walk-in to get started" with the "+ Add walk-in" CTA; the "View full queue →" link remains visible and navigable.
- What happens when a Staff user directly navigates to `/staff` via URL while unauthenticated? → The SPA route guard detects the absence of a valid JWT and redirects to `/login` (SCR-001); the intended destination `/staff` is stored in React Router state and the user is redirected back after successful re-authentication.
