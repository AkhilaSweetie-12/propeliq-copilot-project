# User Story - US_002

## Story ID

- ID Format: US_002

## Story Title

- ASP.NET Core 9 Web API Scaffold with Modular Feature Folders

## Description

- As a developer, I want the ASP.NET Core 9 Web API project scaffolded with modular feature folders, OpenAPI documentation, and structured logging, so that backend feature modules can be built in isolation with a consistent, discoverable architecture pattern from day one.

## Acceptance Criteria

1. **Given** the `/api` project, **When** `dotnet run` is executed, **Then** the API server starts on port 5000 without errors and the Swagger/Scalar OpenAPI UI is accessible at `http://localhost:5000/swagger`, listing all registered endpoints.

2. **Given** the modular feature folder structure (`/api/Features/Booking`, `/api/Features/Intake`, `/api/Features/Clinical`, `/api/Features/Coding`, `/api/Features/Notifications`, `/api/Features/Admin`), **When** a developer adds a new controller class under any feature folder, **Then** ASP.NET Core's controller discovery auto-registers it in the DI container and the endpoint appears in the OpenAPI spec without any manual registration entry in `Program.cs`.

3. **Given** any protected or public API endpoint, **When** 50 sequential requests are made under normal load to a stub health endpoint, **Then** all responses return within 2 seconds p95, establishing the baseline for NFR-001.

4. **Given** a request missing a valid JWT bearer token arrives at a `[Authorize]`-protected endpoint, **When** the request is processed by the auth middleware, **Then** the API returns HTTP 401 Unauthorized with a `WWW-Authenticate: Bearer` header before any handler logic executes; no stack trace is leaked in the response body.

5. **Given** Serilog is configured with the Seq community server sink, **When** any API request is processed, **Then** a structured log entry is written to Seq containing `RequestMethod`, `RequestPath`, `StatusCode`, `ElapsedMs`, and `TraceId` fields, queryable via the Seq UI.

6. **Given** the application starts with a missing or invalid `DATABASE_URL` environment variable, **When** startup runs, **Then** the application throws a descriptive `InvalidOperationException` ("DATABASE_URL environment variable is required") and does not reach the `app.Run()` call; no partial startup occurs.

## Edge Cases

- What happens when a duplicate service registration exists in two feature folders? → The DI container startup throws an `InvalidOperationException` identifying the duplicate; the application does not start, preventing silent service shadowing.
- What happens when the OpenAPI spec generation encounters a controller with a missing `[ApiController]` attribute? → A startup-time Swagger validation step (added to the build pipeline) emits a warning to the Serilog sink; the controller is excluded from the spec to prevent malformed documentation.
- What happens when the API is called with an unsupported HTTP method (e.g., `PATCH` on a `GET`-only endpoint)? → ASP.NET Core returns HTTP 405 Method Not Allowed with an `Allow` header listing valid methods; no unhandled exception is thrown.
