# User Story - US_001

## Story ID

- ID Format: US_001

## Story Title

- React 18 + TypeScript + TailwindCSS SPA Scaffold

## Description

- As a developer, I want the React 18 + TypeScript + TailwindCSS SPA project scaffolded and buildable with a configured Vite dev server and production build pipeline, so that frontend feature development can begin on a consistent, type-safe foundation with no manual tooling setup.

## Acceptance Criteria

1. **Given** an empty `/client` directory in the repository, **When** `npm install && npm run dev` is executed, **Then** the Vite dev server starts without errors and serves the SPA shell at `http://localhost:3000` within 5 seconds, confirming React 18 + TypeScript + TailwindCSS are correctly wired.

2. **Given** the frontend scaffold, **When** `npm run build` is executed, **Then** the TypeScript compiler reports zero type errors and a production bundle is generated in `/client/dist`; the total JavaScript bundle (gzipped) does not exceed 500 KB for the initial shell.

3. **Given** a request to any defined client-side route, **When** the SPA shell is loaded over a standard broadband connection, **Then** Time-to-Interactive (TTI) is below 3 seconds, satisfying NFR-002 from the outset.

4. **Given** TailwindCSS is configured with the design token values from `designsystem.md`, **When** any component applies token-based utility classes, **Then** the compiled CSS reflects the correct colours, spacing, and typography tokens, and `grep` of the production CSS bundle reports zero hard-coded hex colour values outside the Tailwind config file.

5. **Given** React Router is configured with a catch-all fallback route, **When** a user navigates to any path not defined in the route manifest (e.g., `/undefined-page`), **Then** the SPA renders a 404 fallback page without a full browser reload and without a JavaScript console error.

6. **Given** the Vite config includes a proxy rule for `/api`, **When** the backend API is running on port 5000, **Then** a `fetch('/api/health')` call from the SPA is proxied to `http://localhost:5000/api/health` and returns the actual health response without CORS errors.

## Edge Cases

- What happens when `npm install` is run on Node.js below version 20? → The repository `.nvmrc` pins Node.js 20 LTS; the `predev` npm script calls `node --version` and exits with code 1 and a descriptive message (`Requires Node.js >= 20`) if the version is incompatible.
- What happens when the backend API is unreachable and a proxied request fails? → The Vite proxy returns HTTP 502; the SPA renders a non-blocking toast notification ("API unavailable — please retry") using the interaction pattern from EP-010 and does not crash or show an unhandled error boundary.
- What happens when `TailwindCSS` purges a utility class that is dynamically constructed at runtime? → Dynamic class names are added to the `safelist` in `tailwind.config.ts`; a CI lint step (`npx tailwindcss --dry-run`) verifies no critical classes are purged.
